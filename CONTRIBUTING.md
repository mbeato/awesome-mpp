# Contributing to Awesome MPP

Thank you for contributing to the Awesome MPP list!

## Guidelines

### Adding a new item

- Search existing entries to avoid duplicates.
- Use the following format: `[Name](URL) - Description.`
- Keep descriptions concise and factual. End with a period.
- Add new items to the bottom of the relevant section.
- Check that your links are working.

### What qualifies

- **Services**: Must be live and accepting MPP payments (not coming-soon or placeholder).
- **Tools**: Must be functional and related to the MPP ecosystem.
- **SDKs/Libraries**: Must be published (npm, PyPI, crates.io, or GitHub release).
- **Resources**: Must provide genuine value (tutorials, blog posts, documentation).

### What does not qualify

- Abandoned projects (no updates in 6+ months with no stable release).
- Projects that merely mention MPP but don't implement it.
- Duplicate entries for the same project.

### Pull request process

1. Fork the repository.
2. Add your entry in the appropriate section.
3. Ensure the list is alphabetically sorted within each section (where applicable).
4. Submit a pull request with a clear description of what you're adding.
5. One entry per pull request makes reviewing easier but is not required.

### Updating existing entries

If a project has moved, been renamed, or its description is outdated, PRs to update are welcome.

## Code of Conduct

Be respectful. This is a community resource.
